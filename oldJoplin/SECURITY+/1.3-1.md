- cryptographic attacks

- privilege escalation
ganhar acesso administrativo atraves de diversas tecnicas como roubo de senhas, ou vulnerabilidades conhecidas ou (0days) que tornem isso possivel. 

- cross-site scripting(xss)
reflected: user tem que clickar num link
stored: qualquer usuario sofre o ataque, em rede social isso é perigosissimo.
**defesas**:validar input (sanatization), desabilitar js, unao clicar em links de email

- request forgeries
ssrf (pode acontecer no servidor) controla o servidor para roubar infos, eles tem acesso a tudo dentro da rede do ambiente. Capital One bank
csrf (cross site request forgery) ocorre no cliente
o atacante consegue roubar qualquer info do usuario geralemtne secoes para logar em sua conta.
**def:** WAF

- driver manipulation
shimming (middleman) (aquele espaco extra que vc usa para nivelar uma cadeira)
windows compatibility mode
refactoring (malware metamorfico) (adiconar NOS, loops ou codigo inutil)
**def**: nesse caso antivirul baseado em assinatura é inutil. deve ser usado o comportamental ou ML-based

- SSL stripping
atacante deve estar no MITM modificando o trafego do user. Combina o on-path com o downgrade

- race condition
TOCTOU (time of check to time-of-use attack)
vamos dizer que o programa tena uma condicao, e seomente quando ela é acionada, uma acao ocorre no programa. Aqui pode ser feito uma race-condition. Vamos dizer que vc programe um SO para rebootar sempre que ocorra um erro no programa, e assim ele reboot. Só que se o erro for no próprio SO ele vai rebootar infinitamente. MARS rover.
